# Skill: Daily Briefing

Generate a "Morning CEO Briefing" markdown file summarizing the state of the business and personal affairs. Write the output to `/AI_Employee_Vault/Briefings/<YYYY-MM-DD>_Briefing.md`.

## Steps

1. **Read context files:**
   - `AI_Employee_Vault/Business_Goals.md` — revenue targets, active projects, KPIs
   - `AI_Employee_Vault/Dashboard.md` — current system status and recent activity
   - `AI_Employee_Vault/Done/` — count and skim items completed this week
   - `AI_Employee_Vault/Logs/<today>.json` — actions taken today

2. **Calculate metrics:**
   - Tasks completed this week (count files in /Done created in past 7 days)
   - Items still pending in /Needs_Action
   - Items awaiting approval in /Pending_Approval

3. **Identify bottlenecks:**
   - Any item in /Needs_Action older than 24 hours
   - Any item in /Pending_Approval older than 48 hours

4. **Write the briefing** to `AI_Employee_Vault/Briefings/<YYYY-MM-DD>_Briefing.md` using this structure:

```markdown
# Morning Briefing — <DATE>

## Executive Summary
<2-3 sentence overview>

## Tasks
- Completed this week: X
- Pending now: X
- Awaiting approval: X

## Bottlenecks
<list any delayed items>

## Active Projects Status
<status of each project from Business_Goals.md>

## Proactive Suggestions
<any observations or recommendations>

---
*Generated by AI Employee*
```

5. **Update Dashboard.md** — add a link to the new briefing under "Recent Activity".

## Notes
- Create `AI_Employee_Vault/Briefings/` folder if it doesn't exist.
- Keep the tone concise and executive-level — no raw data dumps.
- If there is nothing to report, still generate the briefing with "All clear" summary.
